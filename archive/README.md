# Archive

Historical documentation and deprecated features.

**Purpose:** Preserve context for past decisions and removed functionality.

## Contents

- `business-analysis-ru.md` - Original business analysis (Russian)
- `development-archive.md` - Development notes from early phases
- `docker/` - Deprecated Docker deployment (removed in v1.0)
- `PACKAGE_CONVERSION_PLAN_RU.md` - Original package conversion plan (Russian)

## Note

This directory is for historical reference only. For current documentation, see `docs/`.

## Why Keep Archive?

- **Historical Context:** Understanding why certain decisions were made
- **Evolution Tracking:** See how the project evolved from concept to production
- **Learning Resource:** Learn from past approaches and why they were changed
- **Git History Supplement:** Provides narrative context beyond commit messages

## When to Use

- Researching project history
- Understanding deprecated features
- Learning about removed functionality
- Onboarding new contributors (historical perspective)

## When NOT to Use

- Current development work → see `docs/`
- API reference → see `docs/api_reference.md`
- User guides → see `docs/quickstart.md`
- Contributing → see `CONTRIBUTING.md`
